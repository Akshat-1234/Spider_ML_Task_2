## Base_Task/Level1 - CIFAR-10 Image Classification (ResNet)

A ResNet style CNN built from scratch (not `torchvision.models.resnet`) and trained on CIFAR-10 for 10-class image classification.

**Architecture**

- Stem: single 3x3 conv + BatchNorm + ReLU
- 4 stages of residual blocks (channels: 32 → 64 → 128 → 256), each block has a conv-BN-ReLU-conv-BN path plus an identity/1x1-conv shortcut
- Global average pooling followed by a fully connected layer for the final 10 class output

**Training setup**

- Dataset split: 45,000 train / 5,000 validation (from the CIFAR-10 train set) + 10,000 test
- Augmentation: random crop with padding, random horizontal flip
- Optimizer: SGD (lr=0.1, momentum=0.9, weight decay=5e-4) — Adam was tried first but SGD performed better
- Loss: CrossEntropyLoss
- 20 epochs, best model (highest val accuracy) checkpointed to `best_model.pth`
- Evaluation: test accuracy, classification report, and confusion matrix

**Running it**

Open `ResNet.ipynb` in Jupyter and run all cells top to bottom. CIFAR-10 downloads automatically via `torchvision.datasets.CIFAR10` on first run. Training will use CUDA automatically if available, otherwise falls back to CPU (much slower).

To just evaluate the existing checkpoint without retraining, skip the `train_model(...)` call and run the cells that load `best_model.pth` and evaluate on the test set directly.

The notebook also has a short write-up at the end covering why residual connections help optimization, how skip connections improve gradient flow, the depth/speed tradeoff, and challenges hit during implementation (mainly dimension mismatches in the shortcut path and dataloader worker tuning).

## Applied_ML - Healthcare Information Assistant (RAG)

A RAG pipeline that answers medical questions using MedQuAD and WHO fact sheets as the knowledge base, with safety checks, reranking, confidence scoring, and a faithfulness check before returning an answer. Runs fully locally with Ollama + Llama 3.

Full details, setup, and usage are in [`Applied_ML/Report.md`](Applied_ML/Report.md).

## Setup

```bash
python -m venv .venv
```

Windows

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
.venv\Scripts\Activate.ps1
```

(PowerShell blocks running scripts by default, that first line just allows it for the current terminal session — you'll need to run it again if you open a new terminal.)

Linux / Mac

```bash
source .venv/bin/activate
```

```bash
pip install -r requirements.txt
```

The `Base_Task/Level1` notebook only needs the PyTorch/sklearn/matplotlib stack. The `Applied_ML` code additionally needs Ollama running locally with the `llama3` model pulled — see `Applied_ML/Report.md` for that setup.

## Notes

- `Base_Task/Level1/data/` (the CIFAR-10 tarball and extracted batches) and `Applied_ML/vectorstore/` are regenerated by running the notebook / `build_vectorstore.py` respectively, so they're gitignored rather than committed.
- `best_model.pth` is currently committed to the repo. It's a large binary so if the repo starts feeling heavy, consider moving to Git LFS or dropping it from version control and just regenerating it by re-running the notebook.
